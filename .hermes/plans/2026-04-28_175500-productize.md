# Web3Pay 产品化计划
Date: 2026-04-28

## 当前状态
- 后端: FastAPI + SQLAlchemy async + PostgreSQL + Web3.py
- 前端: Next.js 14 + wagmi + Tailwind
- 部署: Docker Compose + Nginx
- 问题: 路由不完整、CORS全开、无webhook、nonce内存存储、config硬编码

## 修复优先级

### Phase 1: 后端核心 (Critical)
1. **修复路由缺失** — routes.py有auth但缺order CRUD、admin路由
2. **CORS收紧** — `allow_origins=["*"]` → 从环境变量读取
3. **Nonce改Redis** — 内存dict → redis cache（支持多实例）
4. **Webhook系统** — 支付确认后自动POST到webhook_url
5. **Health check** — GET /health返回系统状态
6. **Rate limiting** — slowapi限流防止滥用
7. **修复docker-compose** — REDIS_URL错误

### Phase 2: 前端完善 (Important)
8. **订单列表页** — 展示用户的历史订单
9. **支付详情页** — QR码 + 状态实时刷新
10. **错误处理** — toast通知 + loading states
11. **响应式** — 移动端适配

### Phase 3: 生产加固 (Security)
12. **Rate limit per IP/wallet**
13. **Request validation + sanitization**
14. **HTTPS enforcement**
15. **Admin审计日志完善**

### Phase 4: DevOps
16. **Docker health checks**
17. **日志管理**
18. **CI/CD修复** — 测试通过
19. **部署文档** — README更新
